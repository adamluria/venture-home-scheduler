## Architecture Decisions

- **Suggestion engine uses P(sit) Ã— P(close | sit) rather than a linear score** â†’ keeps the output interpretable as a % and composable with other probabilistic signals later.
- **Proximity score is zip-prefix based (same-zip 1.00 â†’ far 0.72)** â†’ avoids a Google Maps API dependency for v1; upgrade path is `getDistanceMatrix()` helper in `calendarService.js`.
- **Per-rep stats use deterministic jitter** keyed by `hash(repId|leadSource)` â†’ UI ranking is stable across sessions before real SFDC data lands.
- **PWA over native app** â†’ one codebase, no app store review, installable from Safari/Chrome. Rep feedback will decide if we upgrade to Capacitor-wrapped later.

## Architecture Decisions (afternoon)

- **Notification transport via adapter pattern** so mock mode is fully usable offline / in dev; swap to Twilio/SendGrid through a backend endpoint without touching callers.
- **HTML5 drag-drop over `react-dnd`** to avoid a dependency for a single view; `scoreRepSlot` is pure + fast enough to call synchronously on every hover change.
- **Reschedule token is client-side DJB2 hashed** for dev/demo; noted in code that production must switch to server-minted JWTs.
- **Customer/address split heuristic** â€” first digit (street number) or first comma â€” works for ~95% of free-form inputs operators type; edge cases fall through to `NewAppointmentModal` for human review.
- **Mock-data mutation + refresh counter** instead of lifting `mockAppointments` into React state â€” minimizes surface area of this change while still enabling drag-drop reschedule to re-render.
