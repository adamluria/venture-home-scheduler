# Session 2026-04-27

## Components Created

### [Tier 1] StateView.jsx
**File**: `/sessions/great-vibrant-thompson/mnt/venture-home-scheduler/src/components/StateView.jsx`

Grid-based territory view grouped by state. Displays:
- Territory name with color bar indicator
- Appointment count for the week
- Rep count (excludes closers)
- Average close rate across reps
- Average sit rate across reps
- Horizontal bar chart showing appointment load by day of week (7 bars: Mon-Sun)

**Layout**:
- 2 columns on desktop, 1 column on mobile
- Cards sorted by appointment count (descending)
- Territories grouped under state headers

**Design**: Dark theme with amber accent (#F0A830), uses design tokens from theme.js

### [Tier 1] TeamView.jsx
**File**: `/sessions/great-vibrant-thompson/mnt/venture-home-scheduler/src/components/TeamView.jsx`

Expandable accordion view of sales teams (territories) with rep roster. Shows:

**Team Header**:
- Territory name, color indicator, rep count, appointment count, avg close rate
- Click to expand/collapse roster

**Rep Rows** (when expanded):
- Name and position label (RSM/DE/Sr. SC/SC)
- Close rate %
- Sit rate %
- Week appointment count
- Sorted by close rate descending

**Layout**:
- Desktop: table-like grid layout with header row
- Mobile: compact 2-column layout per rep
- Chevron indicator for expand/collapse state

**Design**: Dark theme with territory color bar on left side of team headers

## Design System Applied
- Colors: T.bg, T.surface, T.border, T.accent (#F0A830), T.text, T.muted
- Fonts: Outfit (UI), JetBrains Mono (data/numbers)
- Responsive: useIsMobile hook at 768px breakpoint
- Inline styles only, no CSS files

## Data Integration
- Uses real consultant roster from mockData.js (51 reps)
- Real territory definitions from TERRITORIES object
- Performance stats from getRepOverallStats()
- Appointment counts from getAppointmentsForDateRange()
- Both filter by weekDates and selectedRegions props
